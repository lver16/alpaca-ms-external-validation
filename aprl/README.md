# APRL

This folder contains the scripts to run the APRL (Automated Paramagnetic Rim Lesion) detection pipeline. The pipeline uses MIMoSA to generate lesion candidates from multi-modal MRI images, and then runs the `prlr` package to identify paramagnetic rim lesions.

For more details on the APRL project, refer to the [official prlr repository](https://github.com/carolynlou/prlr).

---

## Workflow

The two scripts must be run in the following order:

```
1. create_inputs_aprl.R   — preprocess MRI images and generate lesion candidates using MIMoSA
2. run_aprl.R             — run APRL detection on the preprocessed inputs
```

---

## Files

- `create_inputs_aprl.R` — Preprocesses multi-modal MRI images and generates lesion candidates using MIMoSA. Uses helper functions from the ALPaCA project (see below).
- `run_aprl.R` — Runs the APRL detection pipeline using the `prlr` package on the outputs of `create_inputs_aprl.R`.
- `00_lesion_helper_functions.R` — Helper functions from the ALPaCA project used by `create_inputs_aprl.R` for lesion labeling and manipulation.
- `00_preprocessing_helper_functions.R` — Helper functions from the ALPaCA project used by `create_inputs_aprl.R` for image preprocessing steps.

**R packages:**

The `create_inputs_aprl.R` script requires a MIMoSA model `.RData` file that must be downloaded separately from the MIMoSA project. See the [MIMoSA repository](https://github.com/avalcarcel9/mimosa) for instructions on how to obtain the model file.

**prlr package:**
The `run_aprl.R` script requires the `prlr` package which must be installed from the [prlr repository](https://github.com/carolynlou/prlr).

## Step 1: Create APRL inputs — `create_inputs_aprl.R`

This script preprocesses multi-modal MRI images (T1, FLAIR, EPI, phase) and generates lesion candidates using MIMoSA. It relies on helper functions from the ALPaCA project (`00_lesion_helper_functions.R` and `00_preprocessing_helper_functions.R`) which must be present in the same folder.

### Input folder structure

Each subject folder must contain the following files:

```
<dataset_root>/
└── <subject_id>/
    ├── T1.nii.gz       # T1-weighted MRI
    ├── FLAIR.nii.gz    # FLAIR MRI
    ├── EPIm.nii.gz     # EPI magnitude image
    └── EPIp.nii.gz     # EPI phase image
```

Additional input files required per subject:
```
<brainmask_root>/<subject_id>_brainmask.nii.gz
<labels_root>/<subject_id>_ses-01_mask-instances.nii.gz
```

**Label encoding expected in instance label masks:**
- `0` = background
- `1000–1999` = PRL lesions
- `>= 2000` = non-PRL lesions

### Usage

```bash
Rscript create_inputs_aprl.R <sublist_file> <dataset_root> <brainmask_root> \
        <labels_root> <out_root> <mimosa_model_file>
```

| Argument | Description |
|---|---|
| `sublist_file` | Path to a text file listing subject IDs, one per line (e.g. `sub-001`). Session suffix `_ses-01` is added automatically. |
| `dataset_root` | Path to the root directory containing subject folders |
| `brainmask_root` | Path to the directory containing brain masks |
| `labels_root` | Path to the directory containing raw instance label masks |
| `out_root` | Path to the root directory where outputs will be saved |
| `mimosa_model_file` | Path to the MIMoSA model `.RData` file |

**Example:**
```bash
Rscript create_inputs_aprl.R subjects.txt /data/dataset /data/brainmasks \
        /data/labels /data/processed /models/mimosa_model.RData
```

**Note:** By default the script reprocesses all subjects even if outputs already exist. If you want to skip subjects that have already been processed (e.g. after a crash on a large dataset), set `skip_exists <- TRUE` directly in the script.

### Outputs

For each subject, the following files are saved in `<out_root>/<subject_id>_ses-01/`:

| File | Description |
|---|---|
| `prob.nii.gz` | MIMoSA lesion probability map |
| `t1_final.nii.gz` | WhiteStripe-normalized T1 |
| `flair_final.nii.gz` | WhiteStripe-normalized FLAIR |
| `epi_final.nii.gz` | Z-score normalized EPI |
| `phase_final.nii.gz` | WhiteStripe-normalized phase |
| `mask_final.nii.gz` | Brain mask in EPI space |
| `prl_truth_mask.nii.gz` | Binary PRL truth mask |
| `lesion_truth_mask.nii.gz` | Binary lesion truth mask (PRL + non-PRL) |
| `prob_50_binary.nii.gz` | MIMoSA probability map thresholded at 0.5 |
| `prob_50_labeled.nii.gz` | Labeled lesion candidates |

## Step 2: Run APRL — `run_aprl.R`

This script runs the APRL detection pipeline using the `prlr` package on the outputs generated by `create_inputs_aprl.R`.

### Usage

```bash
Rscript run_aprl.R <sublist_file> <out_root> [lib_path]
```

| Argument | Description |
|---|---|
| `sublist_file` | Path to a text file listing subject IDs, one per line. Subject IDs must include the session suffix (e.g. `sub-001_ses-01`). |
| `out_root` | Path to the root directory containing preprocessed subject folders (output of `create_inputs_aprl.R`) |
| `lib_path` | (Optional) Path to a custom R library directory if `prlr` is installed in a non-default location |

**Example:**
```bash
Rscript run_aprl.R subjects.txt /data/processed
# With custom library path:
Rscript run_aprl.R subjects.txt /data/processed /custom/R/libs
```

### Outputs

For each subject, the following files are saved in `<out_root>/<subject_id>/aprl/`:

| File | Description |
|---|---|
| `findprls_out.RDS` | Full output object from `prlr::findprls()` |
| `aprl_leslabels.nii.gz` | Labeled lesion mask with PRL predictions |
| `aprl_ria.csv` | Rim intensity asymmetry features per lesion |
| `aprl_preds.csv` | PRL predictions per lesion |
